## 2. 模型架构详解

### 2.1 MTCN Backbone (多模态时序卷积网络)

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                        MTCNBackbone 模型架构                                  │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  输入层                                                                      │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │                                                                        │ │
│  │  音频序列特征组                视频特征组                              │ │
│  │  ┌─────────────┐              ┌─────────────┐                         │ │
│  │  │ mel_mfcc    │              │ headpose    │                         │ │
│  │  │ (T, D_mel)  │              │ (T, D_pose) │                         │ │
│  │  ├─────────────┤              ├─────────────┤                         │ │
│  │  │ vad         │              │ face_beh    │                         │ │
│  │  │ (T, D_vad)  │              │ (T, D_beh)  │                         │ │
│  │  ├─────────────┤              ├─────────────┤                         │ │
│  │  │ ssl_embed   │              │ body_pose   │                         │ │
│  │  │ (T, D_ssl)  │              │ ...         │                         │ │
│  │  └─────────────┘              └─────────────┘                         │ │
│  │                                                                        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│           ▼                  ▼                                               │
│                                                                              │
│  适配层 (GroupAdapter)                                                       │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │                                                                        │ │
│  │  目的: 将不同维度的特征投影到统一的 d_adapter 维度                      │ │
│  │                                                                        │ │
│  │  结构:                                                                  │ │
│  │    LayerNorm(d_in) → Linear(d_in, d_adapter) → GELU → Dropout          │ │
│  │                                                                        │ │
│  │  输出:                                                                  │ │
│  │    audio_adapted: list of (T, d_adapter) tensors                       │ │
│  │    video_adapted: list of (T, d_adapter) tensors                       │ │
│  │                                                                        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│           ▼                  ▼                                               │
│                                                                              │
│  模态融合层 (ModalityFusion)                                                 │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │                                                                        │ │
│  │  目的: 在模态内融合多个特征组                                           │ │
│  │                                                                        │ │
│  │  音频融合:                                                              │ │
│  │    concat(audio_adapted) → Linear(n_audio_groups × d_adapter, d_model) │ │
│  │    输出: (T, d_model)                                                  │ │
│  │                                                                        │ │
│  │  视频融合:                                                              │ │
│  │    concat(video_adapted) → Linear(n_video_groups × d_adapter, d_model) │ │
│  │    输出: (T, d_model)                                                  │ │
│  │                                                                        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│           ▼                  ▼                                               │
│                                                                              │
│  TCN 时序建模层                                                              │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │                                                                        │ │
│  │  膨胀残差卷积网络 (Dilated Residual TCN)                                │ │
│  │                                                                        │ │
│  │  ┌──────────────────────────────────────────────────────────────────┐ │ │
│  │  │  DilatedResidualBlock                                             │ │ │
│  │  │                                                                   │ │ │
│  │  │  输入 x (T, d_model)                                              │ │ │
│  │  │      │                                                            │ │ │
│  │  │      ├──────────────────────────────────┐                        │ │ │
│  │  │      │                                  │                        │ │ │
│  │  │      ▼                                  │ residual                │ │ │
│  │  │  LayerNorm → Conv1d(dilation=2^i)      │                         │ │ │
│  │  │      │                                  │                         │ │ │
│  │  │      ▼                                  │                         │ │ │
│  │  │  GELU → Dropout                        │                         │ │ │
│  │  │      │                                  │                         │ │ │
│  │  │      ▼                                  │                         │ │ │
│  │  │  LayerNorm → Conv1d(dilation=2^i)      │                         │ │ │
│  │  │      │                                  │                         │ │ │
│  │  │      ▼                                  │                         │ │ │
│  │  │  Dropout → (+) ←───────────────────────┘                         │ │ │
│  │  │      │                                                            │ │ │
│  │  │      ▼                                                            │ │ │
│  │  │  mask × output                                                    │ │ │
│  │  └──────────────────────────────────────────────────────────────────┘ │ │
│  │                                                                        │ │
│  │  层数: n_layers (默认6层)                                              │ │
│  │  膨胀率: 1, 2, 4, 8, 16, 32                                            │ │
│  │  核大小: kernel_size (默认3)                                           │ │
│  │                                                                        │ │
│  │  感受野计算:                                                            │ │
│  │    RF = 1 + (kernel_size - 1) × Σ(2^i) for i in [0, n_layers-1]       │ │
│  │    例: 6层, kernel=3 → RF = 1 + 2×(1+2+4+8+16+32) = 127               │ │
│  │                                                                        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│           ▼                  ▼                                               │
│                                                                              │
│  ASP 注意力统计池化层                                                        │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │                                                                        │ │
│  │  Attentive Statistics Pooling (ASP)                                    │ │
│  │                                                                        │ │
│  │  目的: 将时序特征聚合为固定维度的统计特征                               │ │
│  │  结合VAD信号和质量控制信号增强注意力权重                               │ │
│  │                                                                        │ │
│  │  计算过程:                                                              │ │
│  │    1. 计算注意力分数:                                                  │ │
│  │       e = Linear(x) + α × vad_signal + β × qc_quality                 │ │
│  │       e = e.masked_fill(~mask, -inf)                                  │ │
│  │                                                                        │ │
│  │    2. 计算注意力权重:                                                  │ │
│  │       w = softmax(e)                                                  │ │
│  │                                                                        │ │
│  │    3. 计算加权统计量:                                                  │ │
│  │       mean = Σ(w × x)                                                 │ │
│  │       std = sqrt(Σ(w × (x - mean)²))                                  │ │
│  │                                                                        │ │
│  │    4. 输出:                                                            │ │
│  │       z = concat(mean, std)  → (2 × d_model)                          │ │
│  │                                                                        │ │
│  │  α, β: 可学习的权重参数                                                │ │
│  │                                                                        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│           ▼                  ▼                                               │
│                                                                              │
│  最终融合层                                                                  │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │                                                                        │ │
│  │  拼接所有特征:                                                          │ │
│  │    z_audio (2×d_model)                                                 │ │
│  │    z_video (2×d_model)                                                 │ │
│  │    audio_pooled_adapted (d_adapter per feature)                        │ │
│  │    session_embed (d_session)                                           │ │
│  │                                                                        │ │
│  │  总维度: 2×d_model×2 + n_pooled×d_adapter + d_session                  │ │
│  │                                                                        │ │
│  │  MLP融合:                                                               │ │
│  │    Linear(总维度, d_shared) → GELU → Dropout → Linear(d_shared, d_shared)│ │
│  │                                                                        │ │
│  │  输出: session_repr (N, d_shared)                                       │ │
│  │                                                                        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 GroupedModel (参与者级聚合)

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                       GroupedModel 聚合架构                                   │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │                                                                        │ │
│  │  输入:                                                                   │ │
│  │    session_reprs: Backbone输出 (N = B×4, d_shared)                     │ │
│  │    session_valid: 会话有效性掩码 (B, 4)                                │ │
│  │    n_participants: B                                                   │ │
│  │                                                                        │ │
│  │           ▼                                                            │ │
│  │                                                                        │ │
│  │  重塑为会话网格:                                                        │ │
│  │    session_grid = session_reprs.view(B, 4, d_shared)                   │ │
│  │                                                                        │ │
│  │           ▼                                                            │ │
│  │                                                                        │ │
│  │  ┌──────────────────────────────────────────────────────────────────┐ │ │
│  │  │  ParticipantAggregator                                           │ │ │
│  │  │                                                                   │ │ │
│  │  │  三种聚合方法:                                                    │ │ │
│  │  │                                                                   │ │ │
│  │  │  1. mean: 简单加权平均                                            │ │ │
│  │  │     pooled = Σ(valid × session_reprs) / n_valid                  │ │ │
│  │  │     output = proj(pooled)                                        │ │ │
│  │  │                                                                   │ │ │
│  │  │  2. mlp: 加权平均 + MLP                                           │ │ │
│  │  │     pooled = Σ(valid × session_reprs) / n_valid                  │ │ │
│  │  │     output = Linear → GELU → Dropout → Linear(pooled)            │ │ │
│  │  │                                                                   │ │ │
│  │  │  3. attention: 自注意力聚合                                       │ │ │
│  │  │     scores = query(session_reprs)                                │ │ │
│  │  │     scores = scores.masked_fill(~valid, -inf)                    │ │ │
│  │  │     weights = softmax(scores)                                    │ │ │
│  │  │     pooled = Σ(weights × session_reprs)                          │ │ │
│  │  │     output = proj(pooled)                                        │ │ │
│  │  │                                                                   │ │ │
│  │  └──────────────────────────────────────────────────────────────────┘ │ │
│  │                                                                        │ │
│  │           ▼                                                            │ │
│  │                                                                        │ │
│  │  输出:                                                                   │ │
│  │    participant_repr: (B, d_shared)                                     │ │
│  │    session_reprs: (B×4, d_shared)                                      │ │
│  │    session_type_logits: (B×4, 4) - 会话类型分类                        │ │
│  │                                                                        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│  多任务学习:                                                                  │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │                                                                        │ │
│  │  主任务: 参与者级预测                                                   │ │
│  │    loss_main = task_loss(participant_repr, targets)                   │ │
│  │                                                                        │ │
│  │  辅助任务1: 会话级预测                                                  │ │
│  │    loss_session = task_loss(session_reprs[valid], targets_expanded)   │ │
│  │                                                                        │ │
│  │  辅助任务2: 会话类型分类                                                │ │
│  │    loss_type = CrossEntropy(session_type_logits[valid], session_types)│ │
│  │                                                                        │ │
│  │  总损失:                                                                │ │
│  │    loss = loss_main + w1 × loss_session + w2 × loss_type              │ │
│  │                                                                        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

### 2.3 任务预测头

```
┌──────────────────────────────────────────────────────────────────────────────┐
│                          任务预测头架构                                       │
├──────────────────────────────────────────────────────────────────────────────┤
│                                                                              │
│  A1Head (三分类二元分类)                                                     │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │                                                                        │ │
│  │  目标: 预测抑郁(D)、焦虑(A)、压力(S)三种心理状态                         │ │
│  │                                                                        │ │
│  │  结构:                                                                  │ │
│  │    Linear(d_shared, 3)                                                 │ │
│  │                                                                        │ │
│  │  输出:                                                                  │ │
│  │    logits: (B, 3)                                                      │ │
│  │    probs = sigmoid(logits)                                             │ │
│  │                                                                        │ │
│  │  损失函数:                                                              │ │
│  │    BCEWithLogits(logits, targets, pos_weight)                          │ │
│  │                                                                        │ │
│  │  初始化技巧:                                                            │ │
│  │    bias_init = log(p_pos / p_neg)  ← 根据训练集正样本率初始化偏置       │ │
│  │                                                                        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│  A2OrdinalHead (序数回归)                                                    │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │                                                                        │ │
│  │  目标: 预测21个心理评估项目,每个取值0-3                                  │ │
│  │                                                                        │ │
│  │  核心思想:                                                              │ │
│  │    将序数回归转化为多个阈值二元分类问题                                 │ │
│  │    score >= 1? score >= 2? score >= 3?                                │ │
│  │                                                                        │ │
│  │  结构:                                                                  │ │
│  │    Linear(d_shared, 21 × 3)                                            │ │
│  │    reshape → (B, 21, 3)                                                │ │
│  │                                                                        │ │
│  │  输出:                                                                  │ │
│  │    logits: (B, 21, 3)                                                  │ │
│  │    其中 logits[b, i, k] 表示第i个项目分数 ≥ (k+1) 的概率logit          │ │
│  │                                                                        │ │
│  │  解码方法:                                                              │ │
│  │    1. argmax: score = Σ(sigmoid(logits) > 0.5)                         │ │
│  │    2. expectation: score = round(p1 + p2 + p3)                         │ │
│  │    3. monotonic: 强制单调递减概率,然后argmax                            │ │
│  │                                                                        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
│  CORALHead (Consistent Rank Logits)                                         │
│  ┌────────────────────────────────────────────────────────────────────────┐ │
│  │                                                                        │ │
│  │  目标: 改进的序数回归方法,保证阈值单调性                                 │ │
│  │                                                                        │ │
│  │  核心思想:                                                              │ │
│  │    学习一个分数score和可学习的阈值thresholds                           │ │
│  │    logits = score - thresholds                                         │ │
│  │    thresholds通过softplus保证单调递增                                  │ │
│  │                                                                        │ │
│  │  结构:                                                                  │ │
│  │    score_fc: Linear(d_shared, 21)                                      │ │
│  │    raw_thresholds: Parameter(21, 3)                                    │ │
│  │                                                                        │ │
│  │  计算:                                                                  │ │
│  │    scores = score_fc(x)                (B, 21)                         │ │
│  │    spacings = softplus(raw_thresholds)  (21, 3)                        │ │
│  │    thresholds = cumsum(spacings)        (21, 3) ← 单调递增              │ │
│  │    logits = scores.unsqueeze(-1) - thresholds.unsqueeze(0)             │ │
│  │             → (B, 21, 3)                                               │ │
│  │                                                                        │ │
│  └────────────────────────────────────────────────────────────────────────┘ │
│                                                                              │
└──────────────────────────────────────────────────────────────────────────────┘
```

## 5. 关键设计决策

### 5.1 为什么使用TCN而非RNN/Transformer?

```
TCN的优势:
┌─────────────────────────────────────────────────────────────────────────────┐
│  1. 并行计算:                                                               │
│     TCN可以并行处理整个序列,而RNN必须顺序处理                               │
│     训练速度更快                                                            │
│                                                                             │
│  2. 稳定的梯度流:                                                           │
│     残差连接确保梯度可以顺畅反向传播                                         │
│     避免RNN的梯度消失/爆炸问题                                              │
│                                                                             │
│  3. 灵活的感受野:                                                           │
│     通过调整层数和膨胀率,可以精确控制感受野大小                              │
│     适合处理不同长度的时序模式                                              │
│                                                                             │
│  4. 内存效率:                                                               │
│     不需要存储中间隐藏状态                                                   │
│     内存占用更稳定                                                           │
│                                                                             │
│  5. 适合音视频时序数据:                                                      │
│     音视频特征通常具有局部相关性                                             │
│     TCN的卷积结构很好地捕获这种局部模式                                     │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 为什么使用ASP池化而非简单平均?

```
ASP的优势:
┌─────────────────────────────────────────────────────────────────────────────┐
│  1. 注意力加权:                                                             │
│     自动学习重要的时间帧                                                    │
│     避免无效帧(静音、背景等)稀释有效特征                                    │
│                                                                             │
│  2. 结合VAD和质量控制:                                                       │
│     VAD信号: 强制关注语音活动帧                                             │
│     QC信号: 强制关注高质量帧                                                │
│     α,β参数可学习,适应不同数据                                              │
│                                                                             │
│  3. 统计特征:                                                               │
│     输出mean和std,捕获分布信息                                              │
│     比单一平均更丰富                                                         │
│                                                                             │
│  4. 比较:                                                                    │
│     简单平均: 所有帧平等对待,无效帧干扰                                     │
│     ASP: 自动加权,输出统计量,鲁棒性更强                                     │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.3 为什么使用序数回归而非直接分类?

```
序数回归的优势:
┌─────────────────────────────────────────────────────────────────────────────┐
│  1. 保持序数性质:                                                           │
│     分数 0 < 1 < 2 < 3 有序关系                                             │
│     直接分类(4类)会丢失这种序数信息                                         │
│     序数回归保留相邻类别的相似性                                            │
│                                                                             │
│  2. 更好的评估指标:                                                         │
│     QWK(Quadratic Weighted Kappa)考虑序数距离                               │
│     相邻预测(预测1实际2)比远距离预测(预测0实际3)惩罚更小                    │
│     序数回归天然适合QWK                                                     │
│                                                                             │
│  3. 参数效率:                                                               │
│     每个项目只需要3个阈值,而非4个类别                                       │
│     减少参数量,降低过拟合风险                                               │
│                                                                             │
│  4. 解码灵活性:                                                             │
│     argmax: 简单计数                                                        │
│     expectation: 连续期望                                                   │
│     monotonic: 强制单调性                                                   │
│     可以选择最适合数据的解码方法                                            │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.4 为什么使用多任务学习?

```
多任务学习的好处:
┌─────────────────────────────────────────────────────────────────────────────┐
│  1. 参与者级 + 会话级预测:                                                   │
│     主任务: 参与者心理健康预测                                              │
│     辅助任务: 每个会话的预测                                                │
│     共享底层表示,增强泛化                                                   │
│                                                                             │
│  2. 会话类型分类:                                                           │
│     辅助任务: 预测会话类型(A01/B01/B02/B03)                                 │
│     强迫模型学习会话语义差异                                                │
│     有助于区分不同情感状态的会话                                            │
│                                                                             │
│  3. 正则化效果:                                                             │
│     多任务共享表示,防止主任务过拟合                                         │
│     等效于隐式数据增强                                                      │
│                                                                             │
│  4. 梯度互补:                                                               │
│     不同任务的梯度可能互补                                                  │
│     更稳定的训练                                                            │
└─────────────────────────────────────────────────────────────────────────────┘
```
